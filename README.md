# Cyton

**Cyton** — учебный язык программирования и реализация его компилятора/интерпретатора на C++23.

Проект реализует основные этапы обработки исходного кода:

```text
Исходный код
    ↓
Лексический анализ
    ↓
Поток токенов
    ↓
Синтаксический анализ
    ↓
AST
    ↓
Семантический анализ
    ↓
Выполнение через tree-walk interpreter
```

На текущем этапе Cyton выполняет программы напрямую по AST с помощью интерпретатора.

---

## Возможности языка

Язык поддерживает:

* переменные `let` и `var`;
* вывод типов переменных из инициализатора;
* целые числа;
* вещественные числа;
* булевы значения;
* строки;
* массивы;
* структуры;
* функции;
* рекурсивные функции;
* пространства имён;
* синонимы типов;
* арифметические операции;
* логические операции;
* операции сравнения;
* побитовые операции;
* условный оператор `if / else`;
* цикл `while`;
* инструкции `break`, `continue`, `return`;
* встроенные функции `print`, `input`, `exit`, `panic`;
* выполнение инструкций верхнего уровня через hidden main;
* интерактивный режим REPL;
* отладочный вывод токенов;
* отладочный вывод AST.

## Пример программы

```cyton
fun square(int64 x) -> int64:
    return x * x;

fun factorial(int64 n) -> int64:
    if n <= 1:
        return 1;

    return n * factorial(n - 1);

print("Cyton demo")
print(square(7))
print(factorial(5))
```

Реализован hidden main, поэтому top-level можно считать точкой входа

```bash
cmake -S . -B build
```

Запуск:

```bash
./build/cyton
```

## Запуск программы

Запустить файл с программой на Cyton:

```bash
./build/cyton example/hello.cy
```

Например:

```cyton
var x = 3

while x > 0:
    print(x)
    x = x - 1;
```

Результат:

```text
3
2
1
```

## Режим REPL

Если запустить программу без аргументов, откроется интерактивный режим:

```bash
./build/cyton
```

Также REPL можно запустить явно:

```bash
./build/cyton --repl
```

Пример работы:

```text
Cyton REPL  (Ctrl+D / "exit" для выхода)
>> let x = 10
>> print(x)
10
```

Для выхода можно использовать:

```text
exit
```

или:

```text
quit
```

### Переменные

Неизменяемая переменная:

```cyton
let x = 10
```

Изменяемая переменная:

```cyton
var y = 20
```

Переменная с явным типом:

```cyton
let string name = "Cyton"
```

### Условия

```cyton
if x > 0:
    print("positive")
else:
    print("not positive");
;
```

Ветка `else` необязательна:

```cyton
if x == 0:
    print("zero");
```

---

### Циклы

```cyton
var i = 0

while i < 5:
    print(i)
    i = i + 1;
```

Поддерживаются `break` и `continue`:

```cyton
var i = 0

while i < 10:
    i = i + 1

    if i == 3:
        continue;

    if i == 8:
        break;

    print(i);
```

### Функции

```cyton
fun add(int64 a, int64 b) -> int64:
    return a + b;

print(add(2, 3))
```

Функция без возвращаемого значения:

```cyton
fun greet(string name) -> unit:
    print("Hello, " + name);

greet("Cyton")
```


### Пространства имён

```cyton
namespace Math:
    fun square(int64 x) -> int64:
        return x * x;

    fun abs(int64 x) -> int64:
        if x < 0:
            return -x
        else:
            return x;
        ;
    ;
;

print(Math::square(7))
print(Math::abs(-10))
```

---

### Синонимы типов

```cyton
type Meters = int64

let Meters distance = 100

print(distance)
```

Синоним типа не создаёт новый отдельный тип, а является альтернативным именем для уже существующего типа.

## Обработка ошибок

Компилятор/интерпретатор сообщает об ошибках с указанием позиции в исходном коде.

Примеры ошибок:

* использование необъявленной переменной;
* повторное объявление имени в одной области видимости;
* изменение переменной, объявленной через `let`;
* несовпадение типов;
* неправильное количество аргументов функции;
* выход за границы массива;
* деление на ноль;
* некорректное использование `break` или `continue`.

Пример runtime-ошибки:

```cyton
let x = 10 / 0
```

Результат:

```text
runtime error: division by zero
```

## Архитектура реализации

### Лексический анализатор

Лексер преобразует исходный текст в поток токенов.

Он распознаёт:

* ключевые слова;
* идентификаторы;
* числовые литералы;
* строковые литералы;
* операторы;
* разделители;
* комментарии.

Каждый токен содержит:

* тип;
* лексему;
* строку;
* столбец.

---

### Синтаксический анализатор

Парсер реализован методом рекурсивного спуска.

Он получает поток токенов и строит AST.

Для выражений используется разбор по уровням приоритета операторов, поэтому выражение:

```cyton
a + b * c
```

интерпретируется как:

```cyton
a + (b * c)
```

---

### AST

AST представляет программу в виде дерева.

Основные виды выражений:

* литералы;
* идентификаторы;
* унарные выражения;
* бинарные выражения;
* вызовы функций;
* индексирование;
* доступ к полям;
* доступ к элементам пространства имён;
* литералы массивов;
* литералы структур.

Основные виды инструкций:

* объявление `let`;
* объявление `var`;
* присваивание;
* выражение как инструкция;
* `if`;
* `while`;
* `break`;
* `continue`;
* `return`;
* блок.

---

### Семантический анализатор

Семантический анализатор проверяет программу после парсинга.

Он отвечает за:

* разрешение имён;
* проверку областей видимости;
* проверку повторных объявлений;
* проверку типов;
* проверку мутабельности переменных;
* проверку вызовов функций;
* проверку возвращаемых значений;
* проверку корректности `break` и `continue`.

Области видимости реализованы как стек таблиц символов.

---

### Интерпретатор

Интерпретатор выполняет AST напрямую.

Runtime-значения представлены через `std::variant`.

Поддерживаемые runtime-значения:

* `unit`;
* `int64`;
* `float64`;
* `bool`;
* `string`;
* массив;
* структура.

Массивы и структуры хранятся как объекты с динамическим содержимым. Для сохранения value semantics используется копирование значений.

---

## Текущие ограничения

На данный момент проект имеет следующие ограничения:

* нет генерации x86-64 или LLVM IR;
* нет создания отдельного исполняемого файла из `.cy`;
* основная точка входа сейчас — top-level код через hidden main;
* типы `int8`, `int16`, `int32`, `int64` внутри реализации могут сводиться к одному представлению;
* типы `float32` и `float64` внутри реализации могут сводиться к одному представлению;
* тип массивов в системе типов реализован частично;
* флаг `-o` пока не реализован.
